# SFERA LMS — Контекст проекта (Frontend)

Ты работаешь с существующим frontend-проектом системы **SFERA LMS**.

## 1. О продукте

SFERA LMS — это веб-приложение (SaaS), предназначенное для рекрутмента и обучения кандидатов через прохождение курсов.

Компании:

- создают курсы
- отправляют кандидатам ссылку
- получают структурированные данные
- управляют кандидатами

Система заменяет внешние LMS и оптимизирует найм.

---

## 2. Архитектура продукта

- Multi-tenant система (несколько компаний)
- Каждая компания работает в **изолированном Пространстве**
- Пространства полностью изолированы по данным
- Доступ зависит от статуса подписки (Активно / Приостановлено)

---

## 3. Роли пользователей

### Суперадминистратор

- управляет Пространствами компаний
- управляет подписками
- имеет полный доступ

### Администратор компании

- управляет пользователями
- создает и редактирует курсы
- управляет кандидатами
- видит аналитику

### Пользователь (HR)

- смотрит курсы и аналитику
- управляет статусами кандидатов
- не редактирует сущности

### Кандидат

- внешний пользователь
- проходит курс по ссылке
- заполняет анкету
- отправляет отклик
- не имеет полноценного кабинета (в MVP)

---

## 4. Основные сущности

- Пространство компании
- Подписка

---

## 5. Основные сценарии

### Управление компанией

- создание Пространства
- назначение администратора

### Пользователи

- приглашение
- регистрация по ссылке
- управление ролями

### Курсы

- создание курса
- авто-добавление первого урока (анкета)
- добавление уроков
- публикация / черновик
- генерация ссылки

### Кандидат

- переход по ссылке
- заполнение анкеты
- прохождение уроков
- сохранение прогресса
- отправка отклика

### Аналитика

- просмотр кандидатов
- фильтрация / сортировка
- изменение статуса
- просмотр ответов

---

## 6. Границы MVP (ВАЖНО)

В MVP есть:

- Пространства компаний
- роли и доступы
- курсы и уроки
- прохождение по ссылке
- сохранение прогресса
- отклики
- базовая аналитика

В MVP НЕТ:

- личного кабинета кандидата
- интеграций (HRM/ATS)
- сложной аналитики
- мобильного приложения
- кастомизации UI
- мультиязычности

---

## 7. Критичные бизнес-правила

- данные компаний изолированы
- пользователь принадлежит только одному Пространству
- доступ зависит от роли
- при приостановке Пространства:
  - никто не может работать
  - кандидаты не могут проходить курсы

- доступ к курсу только по уникальной ссылке
- один отклик = один курс
- кандидат может проходить несколько курсов

---

## 8. Принципы разработки фронтенда

- проект уже имеет архитектуру → не ломать её
- использовать существующие компоненты
- не создавать дублирующие решения
- не усложнять MVP

---

## 9. UX-требования

Интерфейс должен поддерживать:

- таблицы (курсы, пользователи, кандидаты)
- формы (создание / редактирование)
- состояния:
  - loading
  - empty
  - error
  - доступ запрещён
  - черновик / опубликовано

---

## 10. Ограничения

- только веб
- современные браузеры
- работа с персональными данными (осторожно с логами и UI)
