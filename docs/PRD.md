# SFERA LMS — PRD (Product Requirements Document)

## 1. Цель продукта

Создать LMS-систему для:

- замены стороннего решения
- ускорения найма
- снижения затрат
- структурированной оценки кандидатов

---

## 2. Пользователи

- HR и рекрутеры
- владельцы бизнеса
- руководители
- кандидаты

---

## 3. Ценность продукта

Компания:

- создает курс вместо собеседования
- отправляет ссылку кандидату
- получает структурированный отклик
- быстро принимает решение

---

## 4. Основные модули

### 1. Управление компаниями

- создание Пространства
- управление подпиской
- активация / блокировка

### 2. Пользователи

- приглашение
- регистрация
- роли
- удаление

### 3. Курсы

- создание
- редактирование
- уроки
- черновики / публикация
- ссылка на курс

### 4. Кандидаты

- прохождение курса
- заполнение анкеты
- отклик

### 5. Аналитика

- список кандидатов
- прогресс
- фильтры
- статусы

---

## 5. Эпики (Frontend)

### EPIC 1 — Авторизация и приглашения

- регистрация по ссылке
- логин
- сброс пароля

### EPIC 2 — Пространство компании

- настройки
- статус подписки

### EPIC 3 — Пользователи

- список
- приглашение
- роли
- удаление

### EPIC 4 — Курсы

- создание
- редактирование
- уроки
- публикация

### EPIC 5 — Кандидатский поток

- публичная страница курса
- анкета
- прохождение уроков
- прогресс

### EPIC 6 — Кандидаты

- таблица
- статусы
- комментарии
- ответы

### EPIC 7 — Аналитика

- статистика
- фильтры
- сортировка

---

## 6. UX требования

- простой интерфейс
- минимальный friction для кандидата
- понятные состояния
- быстрые действия

---

## 7. Нефункциональные требования

- безопасность данных
- стабильность
- поддержка браузеров

---

## 8. Критерии успеха

- можно создать компанию
- можно создать курс
- кандидат проходит курс
- отклик сохраняется
- HR управляет кандидатами
- ≥80% сценариев работают без ошибок

---

## 9. Ограничения

- нет мобильного приложения
- нет интеграций
- нет сложной аналитики
- базовый UI

---

## 10. Как работать с задачами

Для каждой задачи:

- определить эпик
- определить роли
- определить сущности
- реализовать UI + логику
- учесть права доступа
- проверить бизнес-правила
